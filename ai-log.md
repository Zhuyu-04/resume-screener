# AI 协作记录
## 总体流程
本项目通过多次与 AI 对话协作完成。我作为开发者提出需求和修改意见，AI 生成代码框架并不断迭代。
## AI 帮助生成的部分
1. **初始脚本骨架**：要求 AI “写一个 Python 脚本，接受简历 PDF 文件夹，根据 YAML 配置评分，输出 HTML 报告”。AI 给出了基于 `pdfplumber`、`PyYAML`、`Jinja2` 的基本结构。
2. **评分核心逻辑**：AI 实现了 `score_dimension` 函数，使用关键词计数和正则加权。
3. **HTML 模板**：AI 生成了内嵌的 Jinja2 模板，包括条件颜色、进度条、表格排序。
4. **异常处理**：AI 按照要求添加了 `try/except` 包裹 PDF 解析，失败时记录错误且不中断流程。
5. **命令行参数**：AI 使用 `argparse` 实现参数解析。
## 我进行的修改及原因
| 修改项 | 原因 |
|--------|------|
| 将硬编码配置移到 `config.yaml` | 需求明确“评分标准要外化为配置”，AI 初始版本将维度写死在脚本中，我重构为加载外部 YAML 文件。 |
| 评分归一化公式调整 | AI 起初直接累加关键词次数，导致某些简历得分超过 1000。我改为归一化到 0-100，并增加 `clip` 逻辑。 |
| 增加 `regex_patterns` 支持 | 仅靠关键词无法准确匹配“5+ years of experience”这类表述，我在配置中增加了正则字段，并修改评分函数。 |
| 排序逻辑移到报告生成前 | AI 最初在 HTML 模板中排序，我改为在 Python 中排序后再渲染，保证 JSON 输出也为有序（虽然 JSON 本身无序，但列表顺序可保留）。 |
| 加入文件标识符字段 | 方便在报告中看到文件名，而非路径。 |
## AI 给出错误结论的实例
**问题**：AI 在编写“工作经验”匹配的正则时，给出了 `\d+\s+years?\s+experience`，测试发现无法匹配“5+ years of experience”或“10+ yrs experience”这类常见表述。
**发现过程**：我使用一份真实样本简历进行实战测试，发现“工作经验”维度得分始终为 0。通过打印匹配结果定位到正则失效。
**纠正方法**：将正则改为 `\d+[+]?\s*years?\s*(of)?\s*experience`，并增加了可选符号和简写，验证后生效。同时我在配置中保留两个匹配字段（关键词与正则），确保即使正则出错，关键词仍能兜底。
**教训**：AI 提供的正则往往过于理想化，实际文本格式多样，必须用真实数据验证并及时校准。
